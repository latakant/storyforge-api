## Product Model

**Name:** StoryForge
**Type:** CMS / Publishing Platform

**Description:** A multi-role blogging platform where Writers create and publish articles, Editors curate and approve submissions, and Readers discover and engage with content.

**Actors**
- Primary: Reader · Writer · Editor · Admin
- Secondary: Email provider (notifications) · Cloudinary (images)

**Core Product Lifecycle**
1. Writer creates Article (DRAFT)
2. Writer submits for review (SUBMITTED)
3. Editor approves → PUBLISHED | Editor rejects → DRAFT (with note)
4. Readers discover and read articles
5. Readers clap (event) / comment (PENDING)
6. Editor/Admin moderates comments (APPROVED / REJECTED)
7. Admin or Writer archives article (ARCHIVED)

**Key Entities:** User · Article · Revision · Clap · Comment · Tag · Follow

**Money:** None in scope

**External Dependencies:** Resend (email) · Cloudinary (images)

## Critical Flows

### Flow 1 — Article Publication
- Trigger: Editor calls `PATCH /api/articles/:id/publish`
- Invariant: slug must be set + unique before status → PUBLISHED; slug is then locked
- `$transaction`: Article.status = PUBLISHED + publishedAt = now()
- Side effect (isolated): notify followers via MailerService BullMQ queue

### Flow 2 — Content Edit (Revision Creation)
- Trigger: Writer calls `PATCH /api/articles/:id/content`
- Invariant: NEVER overwrite existing content — always INSERT new Revision
- `$transaction`: INSERT Revision(articleId, content) only

### Flow 3 — Comment Moderation
- Trigger: Editor calls `/api/comments/:id/approve` or `/reject`
- Invariant: only PENDING → APPROVED/REJECTED; terminal states cannot transition
- Side effect (isolated): notify commenter on approval via queue

### Flow 4 — Clap Recording
- Trigger: Reader calls `POST /api/articles/:id/claps`
- Invariant: INSERT ClapEvent — never UPDATE a counter column
- Clap count = `SELECT COUNT(*) WHERE articleId = :id` at read time

## Risk Register

| Risk | Severity | Domain | Mitigation |
|------|----------|--------|------------|
| Slug collision on publish | HIGH | Articles | `@@unique(slug)` + P2002→409 |
| Clap flooding (spam) | MED | Claps | Rate limit: 50 claps/min/user |
| Comment spam | MED | Comments | PENDING default + 10/hr rate limit |
| Revision storage growth | MED | Content | Index on [articleId, createdAt]; prune after 20 |
| Mailer failure on publish | MED | Mailer | BullMQ + exponential backoff (3×) + dead-letter |
| Large article content | LOW | Content | Postgres handles; monitor at scale |

**Race conditions to guard:**
- Slug: `@@unique(slug)` + P2002 → 409
- Follow: `@@unique([followerId, followingId])` + P2002 → 409
- ArticleTag: `@@id([articleId, tagId])` + P2002 → 409

**External failure modes:**
- Cloudinary: article saves without cover image — not blocking
- Resend: BullMQ retry × 3 → dead-letter queue; publication not gated on email
